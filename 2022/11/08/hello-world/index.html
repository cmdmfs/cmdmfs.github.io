修改及部署：

如果您已经修改了本地的 Hexo 博客源代码，并想要将这些更改推送到远程仓库进行部署，那么可以执行以下步骤：

  1. 使用 hexo generate 命令生成您的博客静态页面。==如果您已经生成过一次，则可以跳过此步骤。==

  2. 然后使用 hexo deploy 命令将博客部署到远程仓库。在执行此命令之前，请确保您已经正确配置了 _config.yml 文件中的 deploy 部分，以便 Hexo 可以将博客上传到正确的位置。==也可以跳过。==

  3. 如果您正在使用 Git 进行版本控制，则可以使用 hexo d -g 命令自动将更改推送到远程仓库。此命令会自动执行上述步骤 1 和 2，并在完成部署后将更改提交到 Git。==（建议这一步可以不挂梯子）==
